## JavaScript

### 1. 说说事件循环（Event Loop）机制

- JS 是单线程语言，通过**事件循环**实现异步。
- 执行栈（Call Stack）负责执行同步代码。
- 异步任务分为**宏任务**（Macro Task）和**微任务**（Micro Task）：
  - 宏任务：`setTimeout`、`setInterval`、`MessageChannel`、I/O
  - 微任务：`Promise.then`、`MutationObserver`、`queueMicrotask`
- 每次宏任务执行完毕后，会清空**全部**微任务队列，再渲染，再取下一个宏任务。

```js
console.log('script start');

setTimeout(() => console.log('setTimeout'), 0);

Promise.resolve()
  .then(() => console.log('promise 1'))
  .then(() => console.log('promise 2'));

console.log('script end');

// 输出顺序：
// script start → script end → promise 1 → promise 2 → setTimeout
```

### 2. `this` 的指向规则

| 调用方式 | `this` 指向 |

### 3. 闭包的概念与应用

**闭包**是指函数能够访问其词法作用域中的变量，即使该函数在其词法作用域之外执行。

常见应用：

- 数据私有化（模块模式）
- 函数柯里化
- 防抖 / 节流

```js
function counter() {
  let count = 0;
  return {
    increment() { count++; },
    getCount()  { return count; },
  };
}

const c = counter();
c.increment();
c.increment();
console.log(c.getCount()); // 2
```

- 属性查找沿原型链逐级向上，直至 `null`。
- `instanceof` 运算符通过原型链判断类型。

## CSS

### 1. BFC（块级格式化上下文）

**触发条件**（满足任一即可）：

- `overflow` 不为 `visible`
- `display: flex / inline-flex / grid / inline-grid / flow-root`
- `position: absolute / fixed`
- `float` 不为 `none`

**作用**：

- 阻止外边距合并（margin collapse）
- 包含浮动子元素（清除浮动）
- 防止文字环绕浮动元素


## 浏览器与网络

### 1. 从输入 URL 到页面展示的过程

1. **DNS 解析**：域名 → IP 地址
2. **TCP 三次握手**：建立连接
3. **发送 HTTP 请求**
4. **服务器处理并返回响应**
5. **浏览器解析 HTML**，构建 DOM 树
6. **解析 CSS**，构建 CSSOM 树
7. **合并为渲染树**（Render Tree）
8. **Layout（回流）**：计算几何信息
9. **Paint（重绘）**：绘制像素
10. **Composite（合成）**：GPU 合并图层，展示画面

---

### 2. HTTP 缓存策略

| 类型 | 响应头 | 说明 |
|---|---|---|
| 强缓存 | `Cache-Control: max-age=N` / `Expires` | 未过期时直接读本地，不发请求 |
| 协商缓存 | `ETag` / `Last-Modified` | 向服务器验证资源是否变化，未变化返回 304 |

**优先级**：`Cache-Control` > `Expires`；`ETag` > `Last-Modified`

---

### 3. HTTPS 握手过程（TLS 1.2 简述）

1. Client Hello：客户端发送支持的加密套件、随机数
2. Server Hello：服务端选定套件、返回证书、随机数
3. 客户端验证证书，生成预主密钥（Pre-master Secret），用服务端公钥加密后发送
4. 双方各自根据三个随机数生成**会话密钥**（对称加密）
5. 后续通信使用会话密钥加密

## React

### 1. Hooks 使用规则及常见 Hook

**规则**：

- 只在**函数组件顶层**调用 Hook，不在条件/循环中调用
- 只在 React 函数组件或自定义 Hook 中调用

**常用 Hook**：

| Hook | 用途 |
|---|---|
| `useState` | 组件状态 |
| `useEffect` | 副作用（请求、订阅、DOM 操作） |
| `useCallback` | 缓存函数引用，避免子组件不必要重渲染 |
| `useMemo` | 缓存计算结果 |
| `useRef` | 访问 DOM / 保存不触发渲染的可变值 |
| `useContext` | 消费 Context |

---

### 2. `useEffect` 依赖数组

```jsx
useEffect(() => {
  // 每次渲染后执行
});

useEffect(() => {
  // 仅挂载时执行（类似 componentDidMount）
}, []);

useEffect(() => {
  // dep 变化时执行
  return () => {
    // 清理函数（类似 componentWillUnmount）
  };
}, [dep]);
```

---

### 3. React 性能优化手段

- `React.memo` 跳过 props 未变化的子组件重渲染
- `useCallback` / `useMemo` 稳定引用
- 列表使用稳定的 `key`
- 懒加载：`React.lazy` + `Suspense`
- 代码分割（Code Splitting）
- 避免在渲染函数中创建大对象/函数

## 工程化

### 1. Webpack 构建流程

1. 读取配置，初始化 `Compiler`
2. 从 `entry` 出发，递归解析依赖（`ModuleGraph`）
3. 使用 **Loader** 转换各类模块（`.ts`、`.scss` 等）
4. 使用 **Plugin** 在特定生命周期钩子中扩展功能
5. 输出 `chunk` → `bundle` 到 `output` 目录

---

### 2. Loader 与 Plugin 的区别

| | Loader | Plugin |
|---|---|---|
| 职责 | 文件转换（源码 → JS） | 打包流程扩展 |
| 运行时机 | 模块被加载时 | Webpack 整个生命周期 |
| 写法 | 导出一个函数 | 导出一个带 `apply` 方法的类 |

## 算法手写

### 1. 实现防抖（debounce）

```js
function debounce(fn, delay) {
  let timer = null;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
```

---

### 2. 实现深拷贝（简版）

```js
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (map.has(obj)) return map.get(obj);

  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);

  for (const key of Reflect.ownKeys(obj)) {
    clone[key] = deepClone(obj[key], map);
  }
  return clone;
}
```

---

### 3. 手写 `Promise.all`

```js
function promiseAll(promises) {
  return new Promise((resolve, reject) => {
    const results = [];
    let completed = 0;

    if (promises.length === 0) return resolve(results);

    promises.forEach((p, i) => {
      Promise.resolve(p).then((val) => {
        results[i] = val;
        if (++completed === promises.length) resolve(results);
      }).catch(reject);
    });
  });
}
```
